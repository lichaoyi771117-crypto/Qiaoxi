**1.3 合同文件**：指用户上传至本系统的 PDF、DOCX 或图片格式的合同文本、协议文件及相关附件。

**1.4 输出报告**：指本系统基于用户上传的合同文件，通过人工智能算法处理后生成的《商业决策报告》及其他分析输出物。

**1.6 中间态数据**：指本系统在处理合同文件的过程中生成的、介于原始文件与最终输出报告之间的所有临时性结构化数据（包括但不限于 JSON 格式的条款树、审查中间结果等）。

**1.7 人工智能模型/AI模型**：指本系统调用的第三方大语言模型（包括但不限于 DeepSeek API），用于执行合同文本分析、法律风险识别、商业逻辑推理等任务。AI 模型**仅接收经过脱敏处理后的合同内容**。

## 第二条 服务内容与范围

**2.1 服务内容**

本系统为用户提供以下商业决策辅助服务：
- （1）合同文件的自动解析与条款结构提取；
- （2）基于中国法律法规数据库（本地 RAG）的法律风险初步标定；
- （3）基于商业咨询方法论（系统动力学解构、多角色私董会分析、后果推演）的商业风险识别与杠杆解重构建议；
- （4）结构化《商业决策报告》的自动生成。

**2.2 服务范围与限制**

- **2.2.1** 本系统的分析范围限定于用户上传的合同文件的**文本层面**。本系统不进行事实核查、不验证合同签署方的真实身份或资质、不对合同标的物进行实地勘查或评估。
- **2.2.2** 本系统的法律风险识别基于本地存储的中国法律法规数据库。该数据库可能存在更新延迟、覆盖不全或个别条文解读偏差的情况。标注为**"【法规待核】"** 的风险项，表示本系统未能在本地法律法规数据库中检索到充分的法律依据，用户应另行核实。
- **2.2.3** **本系统不提供任何形式的正式法律意见、诉讼策略建议、仲裁建议或具有法律约束力的文书。本系统的输出报告在任何情况下均不应被解释为构成律师-客户关系、会计师-客户关系或任何其他专业咨询关系。**
- **2.2.4** 本系统当前版本（v0.1 MVP）为最小可行产品（Minimum Viable Product），其中商业模式解构（State 3）、私董会审计（State 4-5）、李超逸最终决策（State 6）等功能模块仍在开发中，以报告中的 "Phase 2 实现" 标注为准。

## 第三条 数据安全与隐私保护

**3.1 数据本地化存储**

**用户上传的合同文件及所有处理过程中产生的中间态数据，均存储于云南程信霖信息咨询有限公司的本地服务器上。程信霖承诺，未经用户另行书面授权，不会将用户的原始合同文件上传、传输、备份、复制或以任何形式提供至任何第三方云服务平台、外部存储设备或境外服务器。**

**3.2 脱敏处理机制（AI 看不见您的真实信息）**

**这是本系统最核心的安全措施，请用户特别关注：**

在将合同内容发送给 AI 模型（DeepSeek API）进行分析之前，本系统自动执行以下脱敏处理，且该脱敏为**不可逆**操作：

| 信息类型 | 脱敏方式 | 示例 |
|---------|---------|------|
| 自然人姓名 | 部分打码 | "张三" → "张\*"、"李四明" → "李\*明" |
| 公司/机构全称 | 匿名化替换 | "云南程信霖信息咨询有限公司" → "【公司A】" |
| 身份证号 | 部分打码 | "440301199001011234" → "440301\*\*\*\*\*\*\*\*1234" |
| 手机号码 | 部分打码 | "13812345678" → "138\*\*\*\*5678" |
| 银行卡号 | 部分打码 | "6222021234567890" → "6222 \*\*\*\* \*\*\*\* 7890" |
| 统一社会信用代码 | 完全替换 | "91530100MA6XXXXXX" → "[已脱敏-统一社会信用代码]" |
| 精确地址 | 模糊化 | "云南省昆明市五华区XX路XX号" → "云南省\*\*\*（详情已脱敏）" |
| 精确金额 | 量级保留 | "5000万元" → "金额X元（千万级）" |
| 电子邮箱 | 完全替换 | "zhangsan@example.com" → "[已脱敏-邮箱]" |

**程信霖特别声明：AI 模型（DeepSeek API）在整个分析过程中，仅能接触到上述脱敏后的文本内容。AI 模型无法获知原始合同中包含的任何自然人姓名、公司全称、证件号码、联系方式、精确地址或精确金额。**

**3.3 文件自动删除机制**

- **3.3.1** 用户每次使用本系统完成一轮合同审查后，可点击"结束服务"按钮。
- **3.3.2** 点击"结束服务"按钮后，系统将自动启动**15 秒倒计时**。倒计时结束后，系统将执行 **shred 级物理删除**——即对文件存储区域进行覆写后删除，确保文件内容不可通过数据恢复工具恢复。
- **3.3.3** 删除范围包括：用户上传的合同原始文件（PDF/DOCX/图片）、所有中间态 JSON 数据、及本次会话中生成的全部临时文件。
- **3.3.4** 即使系统发生意外崩溃或用户未主动点击"结束服务"，系统亦将在**合同上传之日起 30 天后**自动触发物理删除程序。
- **3.3.5** **不删除的内容**：系统审计日志（仅记录操作时间、操作类型、是否成功等元数据，**不包含合同原文或脱敏后的合同内容**）。审计日志采用 WORM（一次写入多次读取）存储，不可篡改，用于系统安全审计和异常追溯。

**3.4 加密存储**

所有在本地服务器上存储的中间态分析数据（JSON 格式）均使用 **AES-256** 加密算法进行加密存储。解密密钥通过环境变量注入，不硬编码于系统源代码中，不随源代码一起进行版本管理。

**3.5 数据传输安全**

本系统与 AI 模型（DeepSeek API）之间的数据传输采用 **TLS 1.3** 加密协议。程信霖仅向 DeepSeek API 发送**已经过脱敏处理的合同文本片段**，不发送原始合同全文。每次 API 调用完成后，发送的上下文立即释放，不在 DeepSeek 服务器端持久化存储。

**3.6 无人工查阅**

在系统自动处理流程中，程信霖的任何员工、顾问或关联方**均不会主动查阅、复制、截取或转发用户上传的原始合同文件**。仅在以下例外情形下，经用户明确书面授权，程信霖授权技术人员可临时访问已脱敏的数据以排查系统故障：
- （1）用户主动报告系统错误，并书面请求人工介入排查；
- （2）系统触发 HANDOFF_TO_HUMAN 机制，且用户选择同意人工顾问介入。

上述访问将在问题解决后 24 小时内终止，访问记录将写入审计日志。

**3.7 用户数据权利**

用户有权在任何时候：
- （1）在使用过程中查看本系统正在处理的数据的阶段状态；
- （2）点击"结束服务"按钮立即触发数据删除；
- （3）通过本协议第十三条列明的联系方式，向程信霖提交数据查阅、更正或删除的书面请求。

程信霖将在收到书面请求后 **15 个工作日内** 予以响应。
