# Qiaoxi 技术约束白皮书（MVP 版）

**定位**：Claude Code 开发硬约束，非架构设计图

**原则**：能跑起来，不能违反；不过度设计，不预支规模

---

## 1. 六君子物理隔离（强制）

- 六君子（李文鸿、吴慧琼、李军、段海涛、王志坚、李艾熹）必须是 **6 个独立进程/线程**，各自拥有独立 LLM 会话上下文。
- 严禁共享 Memory、KV Cache、隐藏状态或任何中间变量。
- 代码层通过会话 ID 硬隔离；单元测试必须验证：A 君子的输出不会出现在 B 君子的上下文中。

---

## 2. 本地 RAG 技术栈（锁定）

| 组件 | 选型 | 约束 |
|------|------|------|
| 向量数据库 | ChromaDB（本地文件型） | 禁止连接外网向量服务 |
| Embedding 模型 | BAAI/bge-m3（本地加载） | 禁止调用 OpenAI/Cohere 等外网 Embedding API |
| 精排 | BM25（本地实现） | Top-10 → BM25 精排 Top-3 |
| 法规库更新 | 离线脚本替换 | 不自动拉取，人工审核后替换 |

---

## 3. 中间态存储与加密（强制）

- 所有中间态 JSON（`contract_raw.json`、`clause_tree.json`、`jo_legal_review.json`、`private_board_audits.json` 等）存放于本地固定目录，**AES-256 加密存储**。
- 解密密钥由环境变量注入，不硬编码于仓库。
- `client_profile.json` 同享此加密策略。

---

## 4. 推演引擎（非 Agent）

- 推演引擎为纯 Python 函数/脚本，输入 CLD 参数，返回模拟结果。
- **禁止在推演引擎内部调用 LLM 生成推演结论**；仅允许确定性计算与预置规则映射。

---

## 5. State 机实现（禁止 If-Else 堆砌）

- State 0–8 必须使用**有限状态机（FSM）**显式实现。
- 每个状态 `transitions` 必须穷举：合法下一 State、错误回退 State、HANDOFF 触发条件。
- **代码审查红线**：State 流转逻辑若出现超过 3 层嵌套 If-Else，必须重构为 FSM。
