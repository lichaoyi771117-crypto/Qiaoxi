## 二、百察产品能力拆解

根据 PPT 提取内容，"百察"的完整功能矩阵如下：

### 2.1 智能合同起草
- **模版合同起草**：基于企业预设的标准合同模板，生成合同初稿
- **非标合同起草**：支持上传文件进行非标合同起草
- **智能抽取填表**：自动从关联材料中提取关键信息，自动填表

### 2.2 智能合同协商
- **内外部协商发起**：打通内外部协商流程，全程数字化留痕
- **协商过程留痕**：记录协商中所有修改意见、沟通记录
- **协商版本管理**：按照协商轮次管理过程中各个版本

### 2.3 智能合同审查
- **合法合规审查**：检查合同是否违反法律法规约定，保障法律底线不破
- **企业自定义规则审查**：按照企业预设的内部规范要求进行审核，满足个性需求
- **形式审查、主体审查**：基础语病逻辑性审查，验证主体资质

### 2.4 智能合同比对
- **多版本比对**：支持模版、修订版、定稿版、归档件等多个版本比对
- **差异位置溯源**：精准的正文定位溯源，一键溯源，差异清晰可见

### 2.5 合同履约管理（推测）
- PPT 标题含"合同履约"但正文未展开，推测包含履约跟踪、到期提醒等 CLM 标配模块

## 三、产品定位差异：CLM vs 商业决策系统

### 3.1 产品类别根本不同

| 维度 | 百察（百融云创） | Qiaoxi（程信霖咨询） |
|------|-----------------|---------------------|
| **产品类别** | **CLM**（合同生命周期管理系统） | **商业决策辅助系统**（合同审查与决策重构） |
| **核心问题** | "合同怎么管？"——提高管理效率 | "合同能不能签？"——辅助商业决策 |
| **用户角色** | 企业法务/合同管理员 | 商业咨询顾问 / 企业决策者 |
| **使用场景** | 日常合同管理流程（起草→审批→履约→归档） | 关键交易的签约前审查（一次性的深度分析） |
| **交付物** | 审查意见 + 比对标注 + 流程记录 | **商业决策报告**（8章，签/改/拖/退 + 理由 + 重构方案） |

### 3.2 核心价值主张不同

**百察的价值**：让企业合同管理工作流数字化、自动化、提效。
- 从起草到履约的"全生命周期"数字化

- 合同版本管理与流程留痕

**Qiaoxi 的价值**：帮决策者回答"这份合同能不能签"。
- 商业咨询五步法的 AI 化落地
- 从客户立场出发的利益分析
- 多 Agent 独立审计 + 系统动力学推演 + 确定性决策

## 四、合同审查能力的深度差异

合同审查是两者的唯一功能重叠区，但深度完全不同。

### 4.1 审查维度对比

| 审查维度 | 百察 | Qiaoxi |
|---------|------|--------|
| **合法合规审查** | ✅ 法条匹配，底线不破 | ✅ 强制本地 RAG（55,088条/1,132部法律），零外部 API |
| **企业自定义规则** | ✅ 按企业内部规范审核 | ✅ 客户画像（State 0）——从客户立场出发的个性化审查 |
| **形式/主体审查** | ✅ 语病检查 + 主体资质验证 | ❌ 不在此层（不关注语法错误和工商登记核验） |
| **商业风险分析** | ❌ 无 | ✅ State 4 六位评审员串行审计（价值投资 / 风控 / 产业 / 交易 / 运营 / 尾部风险） |
| **商业模式解构** | ❌ 无 | ✅ State 3 CLD（因果回路图）——资金/权力/时间三维建模 |
| **后果推演** | ❌ 无 | ✅ State 5 确定性推演引擎（3/6/12/36 个月风险演变） |
| **决策输出** | ❌ 无（审查结论 ≠ 决策） | ✅ State 6 李超逸四选一（签/改/拖/退）+ L1 六戒律硬编码 |
| **合同重构** | ❌ 无 | ✅ State 8 智能重构（新合同草案 + 框架协议 + 尽调清单） |
| **多 Agent 辩论** | ❌ 无 | ✅ 6 位评审员独立审计 + DebateSynthesizer 辩论合成 |

### 4.2 审查方法论对比

**百察的方法**：
```
合同文本 → 法条匹配（合规检查）→ 企业规则匹配（自定义检查）→ 格式/语病检查 → 审查意见
```
本质是 **规则引擎 + AI 辅助** 的合规审查，单向、单层。

**Qiaoxi 的方法**：
```
合同文本 → 客户画像（利益校准）→ 法规RAG（法律初审）→ CLD解构（商业建模）
       → 六位评审员并行审计（六维独立视角）→ 确定性推演（4时间切片）
       → 三层蒸馏决策（硬编码 + 人格核 + 领域规则）→ 商业决策报告 + 重构方案
```
本质是 **多 Agent + 系统动力学 + 确定性决策** 的全栈商业分析，多向、多层。

## 七、综合评估

### 7.1 是否同类产品？

**结论：是相邻赛道，但不是直接竞品。**

百察与 Qiaoxi 的关系类似于：
- **百察 = Concur（费用管理） vs Qiaoxi = 财务顾问（投资决策建议）**
- **百察 = 电子病历系统 vs Qiaoxi = 专家会诊系统**

两者的关系更接近 **互补** 而非 **替代**：
- 百察解决的是"日常合同管理效率"问题（广度），Qiaoxi 解决的是"关键交易决策质量"问题（深度）
- 百察的用户在签日常低风险合同（NDA、采购订单）时不需要 Qiaoxi
- Qiaoxi 的用户面对重大交易（并购、股权转让、合作框架）时，百察的合规审查远远不够

### 7.2 重叠面分析

唯一的功能重叠在于"合同审查"这一层，但深度不在同一量级：

| 场景 | 百察 | Qiaoxi |
|------|------|--------|
| 日常采购合同合规审查 | ✅ 适用（效率工具） | ❌ 过度（杀鸡用牛刀） |
| 重大并购交易风险评估 | ⚠️ 仅法律合规层面 | ✅ 全维度（法律+商业+人性+推演） |
| 合同条款逐条批注 | ✅ | ✅ |
| 交易结构系统性解构 | ❌ | ✅ |
| 未来风险演变推演 | ❌ | ✅ |
| 决策建议（签/改/拖/退） | ❌ | ✅ |
| 自动生成新合同草案 | ❌ | ✅（State 8） |

### 7.3 差异化优势总结

**Qiaoxi 相对于百察的不可替代性**：

1. **决策输出而非审查输出**：百察告诉你"哪里有问题"，Qiaoxi 告诉你"该不该签"
2. **商业视角而非法律视角**：百察是法务工具，Qiaoxi 是咨询顾问工具
3. **多 Agent 独立审计**：六位评审员六种思维原型，记忆隔离，少数意见保留
4. **系统动力学推演**：CLD 因果回路图 + 确定性推演引擎，揭示条款之间的蝴蝶效应
5. **硬编码安全底线**：L1 六戒律 100% 确定性，不可被 LLM"说服绕过"
6. **合同重构能力**：不是批注原合同，而是生成新的合同草案 + 框架协议 + 尽调清单
7. **客户画像驱动**：审查结论锚定在客户具体利益上，而非抽象的法律条文

**百察相对于 Qiaoxi 的优势**：

1. **合同全生命周期管理**：起草→协商→审批→履约→归档的完整闭环
2. **企业级工作流**：内外部协商数字化留痕 + 版本管理
3. **法务团队协作**：多角色审批流程
4. **规模化客户基础**：8000+ 机构，SaaS 成熟度更高
5. **BR-LLM 自研模型**：可能有更低的 AI 推理成本和更可控的性能表现
6. **部署灵活性**：SaaS 云端，无需本地部署维护

## 八、竞争策略建议

### 8.1 定位话术

面对供应链协会等推广渠道，建议区分：

> "百察是做**合同管理**的——帮法务部门管合同、跑流程。我们是做**商业决策**的——帮老板回答'这份合同能不能签'。两者解决的是不同问题，在客户那里可以同时使用。"

### 8.2 差异化切入

- **不要在他们强的地方打**：不必跟 CLM 功能（合同起草/审批流/版本管理），百察在这块有 8000+ 客户和成熟产品
- **在他们做不到的地方深挖**：决策能力（签/改/拖/退）、推演引擎、六维审计、合同重构——这些都是百察 PPT 中完全没有覆盖的维度
- **定价差异化**：百察是 B2B SaaS 年费（推测数十万/年），Qiaoxi 是按次付费（¥20-50/次），目标客群完全不同

### 8.3 可能的合作空间

百察和 Qiaoxi 的产品能力高度互补：

- 百察管"合同流程"，Qiaoxi 管"合同决策"
- 百察覆盖"所有合同", Qiaoxi 聚焦"关键合同"
- 若百察有 API 接口，Qiaoxi 可以成为其"深度分析插件"或"决策模块"

建议关注百融云创/百察的生态开放程度，探索成为其合作伙伴的可能性。
