## 一、用户画像：我们为谁而做？

### 1.1 市场的断层

中国有 **5,200 万家中小企业**，其中 **98% 以上没有法务人员**。这些企业面临一个真实的困境：

- 📌 **日常经营天天见合同**——采购合同、服务协议、租赁合同、合伙协议、保密协议……
- 📌 **每一份合同都是老板自己看**——没有法务、没有顾问、连代账公司都不管合同
- 📌 **代账会计管报税，不管合同风险**——很多小微企业连专职会计都没有，更别提法务
- 📌 **传统律所门槛高**——一份合同审查动辄数千元、等三五天，对小微企业来说是笔不小的开销

### 1.2 百察的市场 vs 我们的市场

| | 百察（百融云创 / 百融智能） | Qiaoxi（程信霖咨询） |
|------|---------------------------|----------------------|
| **目标客户** | 中大型企业、集团公司（有法务部） | 中小微企业、个体工商户、创业者 |
| **客户特征** | 有完整的法务团队，合同管理是日常流程 | 没有法务，甚至没有专职会计，老板自己做决策 |
| **客户痛点** | 合同太多，审批太慢，版本太乱 | **看不懂合同、不敢做决策、不知道能不能签** |
| **产品解决的问题** | "怎么管合同"——提升合同管理流程效率 | **"能不能签"——帮老板做商业决策** |
| **产品性质** | CLM（合同生命周期管理系统）——管理工具 | **商业决策辅助系统**——决策工具 |
| **使用频率** | 日常高频（每份合同都走流程） | 关键节点使用（签重要合同前深度分析） |
| **类比** | 电子病历系统（医院流程管理） | **专家会诊系统（帮你判断要不要做手术）** |

> **一句话定位**：百察帮法务部"管合同"，Qiaoxi 帮老板"做决策"。两者不是竞品，是互补品。一个企业可以同时用百察管日常合同、用 Qiaoxi 审关键合同。

## 二、功能设计：国内独一份

### 2.1 现有产品都在做什么？

国内合同 AI 产品可以分为三大流派：

| 流派 | 代表产品 | 做什么 | 局限 |
|------|---------|--------|------|
| **CLM 管理流** | 百察、幂律 MeFlow、法大大 | 合同起草→审批→签署→履约的全生命周期数字化 | 侧重流程，不做深度商业决策 |
| **合规审查流** | 幂律 MeCheck、案牍AI、慧审通 | 法条匹配 + 企业规则检查 + 风险点标注 | 输出"风险清单"，不回答"能不能签" |
| **虚拟法务流** | 吾律（幂律）、法保网 | 标准化法务服务（合同审核+法律咨询） | 律师视角，不是老板视角；年费制 |

**共同盲区**：所有产品都在回答"合同有什么风险"——但老板真正想知道的是：**"所以呢？我该不该签？"**

### 2.2 Qiaoxi 的差异：从"风险清单"到"商业决策"

Qiaoxi 是国内唯一一个直接输出**商业决策建议（签/改/拖/退）**的合同审查系统。

| 能力维度 | 幂律 MeCheck | 百察 | 案牍AI | 法智通 | 吾律 | **Qiaoxi** |
|---------|-------------|------|--------|--------|------|-----------|
| 法规合规审查 | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| 企业规则审查 | ✅ | ✅ | ✅ | ❌ | ❌ | ✅（客户画像驱动） |
| 合同全生命周期管理 | ✅ | ✅ | ❌ | ✅ | ❌ | ❌（不做 CLM） |
| **客户画像+利益校准** | ❌ | ❌ | ❌ | ❌ | ❌ | ✅ **独有** |

## 三、技术实现：怎么做到的？

### 3.1 技术架构概要

Qiaoxi 采用一套**自研的九阶段审查管线**，合同从上传到决策输出，经过层层递进的分析：

```
合同上传 → 画像校准（AI 动态追问，精准定位你的立场）
   → 条款智能解析 → 法规检索（纯本地，55,000+ 法条）
   → 商业模式建模 → 六维度独立审计
   → 风险推演（未来 3/6/12/36 个月的走势模拟）
   → 决策输出（签/改/拖/退 + 三句理由）
   → 合同重构（生成你的新合同草案）
```

### 3.2 核心技术难点

以下在不暴露过多实现细节的前提下，说明 Qiaoxi 与同类产品的关键技术差异：

**难点一：多维度独立审计架构**

市面上产品大多是"一个 AI 看一遍给出结论"。Qiaoxi 采用**六路并行审计**——六个具有不同思维模型的独立分析引擎，分别从资本保全、风险不对称、产业链卡位、交易结构、运营控制、尾部风险六个角度独立审查，**彼此记忆不共享**，最后交叉比对共识与分歧。这不是"问 AI 六次"——每个引擎有独立的分析逻辑和专用工具。

**难点二：商业化系统动力学建模**

不是简单标注风险条款。Qiaoxi 从合同中提取**资金流向、权力分配、时间约束**三个维度，构建合同背后的商业因果回路图（Causal Loop Diagram），识别"蝴蝶效应"——改动一条款，可能触发哪些连锁反应。这套方法在商业咨询领域已有成熟实践，但用 AI 实现是首次。

**难点三：确定性推演引擎**

同类产品不做推演，或者用 AI"编"推演结论。Qiaoxi 的推演引擎是**纯数学确定性计算**——AI 被明确禁止参与推演结论的生成。我们模拟合同在未来 3 个月、6 个月、12 个月、36 个月四个时间窗口的风险演变轨迹，算法考虑了增强回路（风险随时间放大）和调节回路（刹车机制）的交互作用。每一份合同的推演结果都是**可复现、可审计**的。

**难点四：多层决策架构**

Qiaoxi 的最终决策不是 AI"一拍脑袋"给的，而是经过三层递进：
- **第一层**：硬编码的六条安全底线（100% 确定性，不可被 AI 绕过）——触发即否决，连 AI 都不能推翻
- **第二层**：注入决策者人格——告诉 AI"你是谁、你怎么思考"
- **第三层**：领域专业规则——针对不同交易类型的定制化判断标准

三层按优先级依次检查，AI 只有在前两层都不触发时才有发言权。

**难点五：合同智能重构**

Qiaoxi 不仅能审查合同，还能**重写合同**。系统根据审查结论，自动判断需要生成哪些文件：新合同草案（始终生成）、防御性框架协议（仅并购/股权转让等复杂交易）、尽调清单（仅资产收购场景）。重构方向可三档选择——最大限度维护我方、公平中性、兼顾各方便于达成。每份新合同的关键修改处附有**重构说明注释**，讲清楚改了哪里、为什么改。

**难点六：零外部 API 的法规检索**

55,088 条中国法律法规全部本地化存储和检索，不依赖任何外部 API。数据来源为中国司法部公开的法律法规数据库，经过自研清洗管线（去废法、去重复、条款级切分），确保分析中引用的每一条法条都是现行有效的。

### 3.3 数据安全设计

合同文件上传后，AI 接触到的**不是原文，而是脱敏后的文本**。系统在上传瞬间自动识别并湮灭 8 类敏感信息——身份证号、手机号、银行卡、公司全称、自然人姓名、地址、精确金额、统一社会信用代码——替换为匿名化标记。分析完成后，所有中间文件经过安全覆写后删除。全程有 WORM 审计日志（只追加不修改，每条带防篡改签名）。

## 四、收费模式：按次付费，合作分账

### 4.1 为什么是"按次"？

市面上同类产品的收费模式无一例外是**年费/会员制**：

| 产品 | 收费模式 | 参考价格 |
|------|---------|---------|
| 百察（百融云创） | SaaS 年费 | 推测数十万/年（含 CLM 全模块） |
| 幂律 MeCheck | 定制报价 | 中大型企业年费制 |
| 法保网 | 会员年费 | 1,000-3,000 元/月（小微），5,000-10,000 元/年 |
| 智合同 | 年费 + 模块拆分 | 合同审查模块约 10,000 元/月 |
| 案牍AI | 按合同量 | 按量阶梯，适合合同量波动大的企业 |
| 吾律（幂律） | 年费 | 3999 元/年起 |

**按次收费对中小微企业意味着什么？**

- 🔑 **零门槛**：不需要承诺一年，你今天有一份合同要看，就付一次的钱
- 🔑 **不浪费**：小微企业一年可能只有三五份重要合同需要深度审查，没必要全年订阅
- 🔑 **算得清**：每次多少钱清清楚楚，老板一看就懂，不用算"年化成本"

### 4.2 Qiaoxi 的定价结构

| 服务层级 | 服务内容 | 价格 |
|---------|---------|------|
| **基础审查** | 8 章商业决策报告（Markdown + Word） | ¥20 / 次 |
| **高级重构** | 基础审查 + 新合同草案 + 框架协议 + 尽调清单 | ¥30 / 次 |
| **一次完整咨询** | 基础 + 重构全包 | ¥50 / 次 |

> 对比：传统律所一份合同审查 2,000-5,000 元，等 3-5 天。Qiaoxi：¥20-50，等 5 分钟。

### 4.3 合作推广模式（灵活分账）

Qiaoxi 的推广体系设计为**开放合作模式**，与推广方深度利益绑定：

| 合作模式 | 说明 | 适用场景 |
|---------|------|---------|
| **推广分账** | 推广方带来客户，消费金额按比例分账 | 协会、商会、园区、代理记账公司 |
| **冠名合作** | 推广方冠名——"XX 协会推荐·合同智能审查系统"，程信霖品牌隐名 | 有品牌影响力的渠道 |
| **联合开发** | 双方联名——"XX 协会 & 程信霖咨询 联合出品"，双方品牌并列 | 深度战略合作 |
| **白标嵌入** | 推广方自有品牌呈现，系统底层由 Qiaoxi 驱动 | 有自有客户池的服务商 |

**分账逻辑**：推广方拥有专属授权码前缀，消费自动归属到推广方账户，按约定比例（建议推广方 30-40%）定期结算。

### 4.4 为什么推广方愿意推？

- 💰 **零成本增收**：不需要自己做产品，拿现有渠道变现
- 💰 **提升客户粘性**：帮会员解决合同审查痛点，增加协会/园区服务价值
- 💰 **无库存无售后**：AI 系统自动服务，推广方只负责引流和收钱
- 💰 **可以挂自己的名字**：冠名或联合开发，提升推广方自身品牌形象

## 五、市场时机

### 5.1 为什么是现在？

1. **5,200 万中小企业的法务真空**：98% 没有法务人员，这是一个巨大的空白市场
2. **AI 降低了服务成本**：过去只有大企业用得起的"专业合同审查"，现在可以 ¥20/次提供
3. **老板习惯在移动端做决策**：微信发一份合同、一分钟后收到报告——体验比"等律师三天"好太多
4. **经济下行期企业更谨慎**：每一分钱都要花在刀刃上，每一份合同都要看清楚了再签
5. **赛道尚无同类**：国内没有第二个产品把"合同审查"升级为"商业决策"

### 5.2 竞争壁垒

| 壁垒类型 | 说明 |
|---------|------|
| **方法论壁垒** | 商业咨询五步法 + 系统动力学建模，不是纯技术能复制的 |
| **数据壁垒** | 55,088 条中国法律法规清洗数据库，自研清洗管线 |
| **架构壁垒** | 六 Agent 独立审计 + 确定性推演 + 三层决策蒸馏 |
| **合规壁垒** | 全链路 PII 脱敏 + AES-256 + WORM 审计日志 |
| **商业壁垒** | 推广分账 + 冠名/联合开发模式，渠道利益深度绑定 |
